What These Buttons Mean

🔎 Check One By One

The computer starts at the first item and keeps checking until it finds the match.

✂️ Cut Search In Half

After sorting the list, the computer keeps splitting the remaining choices in half to find the answer faster.

🔍 Deep Search

This tries one route for a long time before backing up and trying another.

🌊 Shortest Steps

This spreads out evenly from the start and is good for finding the path with the fewest steps.

💰 Cheapest Route

This pays attention to expensive areas and tries to find the route that costs the least overall.

🏃 Fast Guess

This rushes toward the goal based on what looks close, even if that guess turns out to be wrong.

⭐ Best Route

This balances distance so far with a smart guess about what looks promising next.

📉 Find Low Spot

This keeps moving downhill to find a low point, like a ball rolling into a valley.

🌡️ Escape Trap

This sometimes allows a worse move now so it can avoid getting stuck in a bad place later.

🧬 Evolve Answers

This tries many possible answers, keeps the stronger ones, and mixes them to make better ones.

📖 Glossary — Key Terms Explained
Algorithm
A step-by-step recipe a computer follows to solve a problem.
Heuristic
A smart guess that helps an algorithm decide which option to try next, even without perfect information.
Node
A single point or square on the grid. The algorithm explores nodes to find a path.
Frontier / Open Set
The list of nodes the algorithm knows about but hasn't explored yet.
Explored / Closed Set
Nodes the algorithm has already visited and evaluated.
Optimal Path
The best possible route — shortest distance or lowest cost — between start and goal.
Local Minimum
A valley that looks like the lowest point nearby, but a deeper valley exists somewhere else.
Global Minimum
The absolute lowest point on the entire surface — the best possible answer.
Gradient
The slope of a surface at a point. Gradient descent follows the slope downhill.
Terrain Cost / Weight
A number that says how hard it is to cross a square. Higher cost means the algorithm prefers to go around.
Mutation
A small random change to a candidate solution, used by genetic algorithms to explore new possibilities.
Population
A group of candidate solutions that a genetic algorithm tests and evolves each generation.
