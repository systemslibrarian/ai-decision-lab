Understanding the Algorithms

🔎 Linear Search

Scans every element one by one until the target is found. Runs in O(n) time. Simple but inefficient for large datasets.

✂️ Binary Search

Requires a sorted array. Halves the search space each step for O(log n) time — exponentially faster than linear on large data.

🔍 Depth-First Search

Explores as deep as possible before backtracking. Fast but does not guarantee the shortest path. Great for maze exploration.

🌊 Breadth-First Search

Explores all neighbors level by level. Guarantees the shortest path on unweighted grids, but ignores terrain cost.

💰 Uniform-Cost Search

Expands the cheapest-cost node first using a priority queue. Guarantees optimal paths on weighted graphs — like Dijkstra's algorithm.

🏃 Greedy Best-First

Expands the node closest to the goal (by heuristic). Very fast but not optimal — can be misled by obstacles and terrain.

⭐ Weighted A*

Uses actual cost + heuristic guidance to find optimal paths through weighted terrain. The gold standard for informed search.

📉 Gradient Descent

Iteratively descends a continuous loss surface to find a local minimum. Powers machine learning, control, and trajectory optimization.

🌡️ Simulated Annealing

A metaheuristic inspired by metallurgy. Accepts worse solutions early (at high temperature) to escape local minima, then progressively refines.

🧬 Genetic Algorithm

Evolves a population of candidate solutions via selection, crossover, and mutation. Great for complex, multi-modal optimization landscapes.
